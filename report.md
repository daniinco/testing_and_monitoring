Поправим 3 проблемы:
в файлике features будем выбрасывать ошибку если одного из нужных полей нет
а в файле app кидаем 422 если упало из-за validation error как раз в предобработке(типа тоже не все фичи)
в файлике app будем писать понятную ошибку если модель упадет при предсказании вероятности
в файлике model добавим понятное сообщение если run_id не передали
в файлике model выбрасываем ошибку если у незагруженной модели вызывается features

тесты модели:
test_model_initial - проверяем что класс модельки инициализируется пустым
test_feat_not_loaded и test_empty_run_id - проверка правильной отработки исправлений для файла model описанных выше
test_random_forest - на примере одного из ранов проверяем что модель грузится и прогоняется нормально.

тесты предобработки:
test_all проверяет создание датасета со всеми фичами по запросу
test_needed проверяет создание датасета только с заданными фичами
test_missing проверяет логику падения при отсутствии нужной фичи в запросе описанную выше

тесты хендлеров
test_health - тестим health
test_predict_success - проверяем что предикт работает
test_missing - предикт правильно падает когда в запросе нет нужного поля
test_invalid - редикт правильно падает когда в запросе неправильный тип поля
test_update_model проверяет апдейт модели
